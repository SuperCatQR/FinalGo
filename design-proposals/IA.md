# FinalGo 站点信息架构（IA）— CHO-94 设计先行

> 本文档对应 PRD「功能一：站点信息架构图」。页面数与类型**以代码实测为准**，已校正描述里的「3 类页面」过期说法。
> 实测口径（`site/` 目录）：**5 类页面 / 54 专业详情 / 19 课程详情（含 15040-migration）/ 1 份样式**。

## 1. 页面树（Page Tree）

```
/  首页 (site/index.html)                                [类型①]
│   └─ 政策核心数字 · 数据源效力链 · 快速入口
│
├─ /majors/  专业索引页 (site/majors/index.html)         [类型②]
│   │   └─ 54 行专业表：序号 / 代码 / 名称 / 层次 / 页面入口(含「建设中」状态)
│   └─ /majors/<代码>-<英文名>/  专业详情页 × 54          [类型③]
│           例：/majors/030101K-law/
│           └─ 计划档案 · 毕业要求 · 课程优先级 · 新旧计划顶替 · 来源
│
└─ /courses/  课程索引页 (site/courses/index.html)        [类型④]
    │   └─ 现行课程组 + 历史存档组（已停用标签）
    └─ /courses/<课程号>/  课程详情页 × 19                [类型⑤]
            例：/courses/15044/  （含 /courses/15040-migration/ 顶替页）
            └─ 见 §3 课程详情区块清单
```

共享层：`site/assets/course.css`（单文件，无设计 token / 组件层）→ 本期原型用 `design-proposals/assets/base.css` 的 token 契约替代，作为「顶模板」骨架雏形。

## 2. 五类页面一览

| # | 页面类型 | 路由 | 数量 | 本期产出原型 | 跳转入口 |
|---|---|---|---|---|---|
| ① | 首页 | `/` | 1 | ✅ 是 | → 专业索引 / 课程索引 / 部分专业·课程直链 |
| ② | 专业索引页 | `/majors/` | 1 | ⬜ 否（复用③样式） | 首页 → 此页 → 54 专业详情 |
| ③ | 专业详情页 | `/majors/<代码>-<英文名>/` | **54** | ✅ 是（代表「专业页」） | 专业索引 → 此页 → 关联课程详情 |
| ④ | 课程索引页 | `/courses/` | 1 | ⬜ 否（复用⑤样式） | 首页 → 此页 → 19 课程详情 |
| ⑤ | 课程详情页 | `/courses/<课程号>/` | **19** | ✅ 是（代表「课程页」） | 课程索引 / 专业详情 → 此页 |

> 负责人原话只点名「首页 + 专业页 + 课程页」三类代表页出原型，故索引页（②④）本期不单独出稿；其样式在选型后由详情页（③⑤）的 token 复用即可，视觉保持一致。

## 3. 课程详情页区块清单（以 15044 为样板，原型骨架须容纳）

实测 15044 含 16 个 `<h1/h2>` 区块，按顺序：

1. 标题（课程名 + 课程号）
2. 政策与时效
3. 考纲概览
4. 教材信息
5. 章节知识树（折叠）
6. 高频概念表 ← **空态样板**：30 项已录入，考频待统计
7. 题型与答题模板
8. 真题解析
9. 考期索引
10. 新旧课程顶替
11. 来源与引用
12. AI 生成声明
13. 相关课程链接
14. 适用专业清单
15. 版本历史
16. 人工审核清单

原型课程页演示了其中代表性区块 + **高频概念表空态**，证明骨架能容纳"有数据 / 暂无数据"两态。

## 4. 状态标注（不虚构页面数量）

- 专业索引 54 行中，多数标 `建设中`（state-chip--transition）；仅法学 / 汉语言文学 / 计算机 / 会计学等少数为完成态。IA 与原型如实保留「建设中」占位，不渲染推测内容。
- 课程索引分「现行课程」与「历史存档」（03708 / 03709 已停用），原型课程页保留"旧课已停用"提示。
