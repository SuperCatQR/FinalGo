### 考试范围

`13003 数据结构与算法` 的学习目标是让考生理解常用数据结构的逻辑结构、存储结构、基本操作、算法设计思想和复杂度分析方法，并能在笔试场景中读懂伪代码、判断边界条件、选择合适结构解决问题。出版社公开目录显示，本书由“自学考试大纲”和 8 个教材章节构成：导论、线性表、栈与队列、数组/广义表/字符串、树和二叉树、图、内部排序、查找。

### 第 1 章 导论

| 编号 | 知识点 | 考核层级 | 审查重点 |
| ---: | --- | --- | --- |
| 1 | 数据、数据元素、数据项、数据对象 | 识记 | 术语层级不可混淆 |

| 5 | 算法特性：有穷性、确定性、可行性、输入、输出 | 识记/理解 | “0 个或多个输入、1 个或多个输出”需核验教材表述 |

导论是全课的评价尺。学习线性表、树、图、排序、查找时，不只记“怎么做”，还要能说明为什么该结构在某类操作上更合适。例如顺序表随机访问为 `O(1)`，但中间插入/删除通常要移动元素；链表插入/删除结点本身可为 `O(1)`，但查找位置常需 `O(n)`。

## 题型与答题模板

> 本区块 ⭐✋——模板为原创方法论，不含第三方原题全文。涉及伪代码和复杂度的内容属于高风险，须 PR 审查团队核验后发布。

### 一、概念辨析题

**题型特征**：围绕术语定义、结构性质、算法特征和适用条件设置单选、填空或判断。常见考点包括逻辑结构/存储结构、稳定排序、哈希冲突、树的度、图的连通性、复杂度阶等。

**答题结构框架**：
1. 先定位术语所属章节，判断是“定义题”还是“性质题”。
2. 对比相近概念：例如线性结构与非线性结构、栈与队列、邻接矩阵与邻接表、稳定与不稳定排序。
3. 注意限定词：最好/平均/最坏、成功/不成功、有向/无向、稠密/稀疏、顺序/链式。

**评分要点**：概念边界清楚；限定词不遗漏；不把某种实现方式当成逻辑结构本身。

### 二、结构操作过程题

**题型特征**：给出一个操作序列，要求写出栈、队列、链表、树、图、排序或查找结构的中间状态和最终结果。

**答题结构框架**：
1. 画出初始结构，标清指针、下标或访问标记。
2. 每执行一步操作，都写出关键状态变化。
3. 对栈/队列题，明确栈顶、队头、队尾；对链表题，明确前驱、当前结点和后继；对排序题，明确“第几趟”的定义按教材口径。
4. 最后给出最终结构，并检查元素个数是否守恒。

**评分要点**：中间步骤可追踪；边界情况处理正确；最终结果与操作序列一致。

### 三、算法分析题

**题型特征**：给出伪代码或算法描述，要求判断功能、写出时间复杂度、空间复杂度或改进思路。

**答题结构框架**：
1. 找循环层数、递归规模变化和主要操作。
2. 区分输入规模变量，例如 `n` 表示元素数，`m` 表示模式串长度，`e` 表示边数。
3. 写复杂度时说明口径：最好、平均、最坏或特定输入。
4. 若算法使用递归、栈、队列、辅助数组，要单独写空间复杂度。

**常见复杂度口径（待教材核验）**：

| 算法/操作 | 时间复杂度 | 空间复杂度 | 注意 |
| --- | --- | --- | --- |
| 顺序表按下标访问 | `O(1)` | `O(1)` | 前提是顺序存储 |
| 顺序表中间插入/删除 | `O(n)` | `O(1)` | 主要代价是移动元素 |
| 单链表按值查找 | `O(n)` | `O(1)` | 不能随机访问 |
| 二叉树遍历 | `O(n)` | `O(h)` | `h` 为递归栈高度，最坏 `O(n)` |
| 邻接矩阵 DFS/BFS | `O(n^2)` | `O(n)` | 需扫描矩阵行 |
| 邻接表 DFS/BFS | `O(n+e)` | `O(n)` | `e` 为边数 |
| 折半查找 | `O(log n)` | `O(1)` | 需有序顺序表 |

### 四、算法设计题

**题型特征**：要求选择合适的数据结构并写出算法思想、伪代码或步骤。例如“判断括号是否匹配”“按层次遍历二叉树”“求图的连通分量”“在有序表中查找元素”。

**答题结构框架**：
1. 说明数据结构选择理由：为什么用栈、队列、树、图或哈希表。
2. 写输入、输出和前置条件。
3. 用结构化伪代码表达主要步骤，避免只写自然语言。
4. 分析时间复杂度和空间复杂度。
5. 补充边界情况：空结构、单元素、重复元素、非法输入。

**示例方法论：括号匹配（不绑定具体试题）**：

- 使用栈保存尚未匹配的左括号。
- 从左到右扫描字符：遇到左括号入栈；遇到右括号时，若栈空则失败，否则检查栈顶是否为对应左括号。
- 扫描结束后，栈空表示全部匹配；栈非空表示存在未匹配左括号。
- 时间复杂度 `O(n)`，空间复杂度最坏 `O(n)`。

### 典型题解析思路

**典型题 1：顺序表与链表的选择**

常见考查形式：给定应用场景，要求判断更适合顺序表还是链表，并说明理由。

解题路径：
1. 看是否需要按下标频繁访问。若需要，顺序表支持 `O(1)` 随机访问。
2. 看是否频繁在中间插入/删除。链表在已知前驱结点时可 `O(1)` 修改指针，但查找位置仍可能 `O(n)`。
3. 看空间是否连续、规模是否容易变化。顺序表需要连续空间，扩容有成本；链表每个结点有额外指针域。
4. 小结时不要绝对化，应结合操作比例和存储约束。

**典型题 2：二叉树遍历序列还原**

常见考查形式：给出前序 + 中序或后序 + 中序，要求还原二叉树或写出另一种遍历序列。

解题路径：
1. 前序首元素或后序末元素是根。
2. 在中序序列中定位根，左侧为左子树，右侧为右子树。
3. 按左右子树规模切分另一条序列，递归还原。
4. 检查是否存在重复关键字；若关键字不唯一，需题目另给区分条件。

**典型题 3：图的最短路径适用条件**

常见考查形式：给出带权图，要求选择 Dijkstra、Floyd 或其他算法，并计算最短路径。

解题路径：
1. 判断是单源最短路径还是多源最短路径。
2. 检查边权是否非负。若存在负权边，Dijkstra 不适用。
3. 按算法维护距离数组和前驱数组，逐步记录中间状态。
4. 最后回溯前驱得到路径，并写出总权值。

**典型题 4：排序算法性质比较**

常见考查形式：判断某排序算法是否稳定、是否原地、最好/平均/最坏复杂度。

解题路径：
1. 先判断算法类别：插入、交换、选择、归并、分配。
2. 稳定性看相等关键字是否可能跨越改变相对顺序。
3. 时间复杂度写明口径：快速排序平均 `O(n log n)`、最坏 `O(n^2)`；直接插入排序最好可 `O(n)`；归并排序通常 `O(n log n)`。
4. 空间复杂度关注递归栈、辅助数组或桶。
