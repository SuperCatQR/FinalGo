1. 审题——明确要求填写的概念/术语/类名/方法

**完整示例**（取自考纲 §Ⅳ.附录题型示例）：

> **题目概要**：Java 中除 ________ 之外的所有类均有一个且只有一个父类。

> **参考答案**：Object

> **参考答案**：Java 中的基本数据类型包括：（1）布尔类型；（2）字符类型；（3）整型；（4）浮点型。

> **解题路径**：按考纲第 2 章「简单应用」知识点，Java 基本数据类型分四类——布尔型（boolean）、字符型（char）、整型（byte/short/int/long）、浮点型（float/double）。分四点作答即可。

### 四、程序改错题（简单应用为主）

**题型特征**：给出带行号的程序代码，要求指出错误行号并改正。覆盖简单应用（Ⅲ）为主。考查对语法规则、数组、循环、类型等细节的掌握。

**答题结构框架**：
1. 通读程序——明确程序功能
2. 逐行检查——重点查声明语法、数组下标越界、循环边界、类型匹配
3. 写明错误行号 + 改正后的代码

**完整示例**（取自考纲 §Ⅳ.附录题型示例，代码可编译运行核验）：

> **题目概要**：请指出错误所在行号并改正。
>
> ```java
> (1)  public class ArrayTest {
> (2)      public static void main(String args[]) {
> (3)          int array[2] = { 1, 2, 3 };
> (4)          for (int ct = 0; ct <= 3; ct++)
> (5)              System.out.printf("%d    %d\n", ct, array[ct]);
> (6)      }
> (7)  }
> ```
>
> **参考答案**：第 3 行和第 4 行有错。
>
> - 第 3 行应改为：`int array[] = { 1, 2, 3 };`（Java 声明数组时方括号内不能写长度，长度由初始化值个数决定）
> - 第 4 行应改为：`for (int ct = 0; ct <= 2; ct++)`（数组有 3 个元素，下标 0–2；原条件 `ct <= 3` 会越界）
>
> **解题路径**：Java 数组声明语法是 `类型[] 名` 或 `类型 名[]`，方括号内**不得**写长度；访问下标从 0 开始，长度为 3 的数组最大下标为 2。

### 五、程序分析题（简单应用/综合应用）

**题型特征**：给出程序，要求写出运行结果。覆盖简单应用（Ⅲ）和综合应用（Ⅳ）。考查对运算、流程控制、类型转换、方法调用的追踪能力。

**答题结构框架**：
1. 通读程序——确定输入与流程
2. 逐步模拟——记录变量取值变化
3. 注意输出格式——println/printf 的格式串与拼接结果

**完整示例**（取自考纲 §Ⅳ.附录题型示例，代码可编译运行核验）：

> **题目概要**：请写出程序的运行结果。
>
> ```java
> class Eple {
>     public static void main(String args[]) {
>         int x, y;
>         x = 61;
>         y = 120;
>         System.out.println("The result is " + (x - y));
>     }
> }
> ```
>
> **参考答案**：`The result is -59`
>
> **解题路径**：`x - y = 61 - 120 = -59`；`+` 在字符串后为字符串拼接，输出 `The result is -59`。注意括号 `(x - y)` 先于字符串拼接计算，否则会被解析为字符串连接。

### 六、程序设计题（综合应用为主）

**题型特征**：按需求编写完整的类或方法。侧重综合应用（Ⅳ）——运用面向对象方法设计程序解决应用问题。

**答题结构框架**：
1. 审题——明确要定义的类、属性、方法
2. 设计类结构——成员变量、构造方法、getter/setter
3. 编写代码——遵循 Java 语法与命名规范
4. 自检——确保可编译、逻辑正确

**完整示例**（取自考纲 §Ⅳ.附录题型示例，代码可编译运行核验）：

> **题目概要**：编写程序，为客户定义一个类，该类包含 3 个属性，分别表示客户的姓名、地址和电话，并定义读取和设置这些数据的方法。
>
> **参考答案**：
>
> ```java
> class Customer {
>     String name, address, telephone;
>     String getName() {
>         return name;
>     }
>     void setName(String name) {
>         this.name = name;
>     }
>     String getAddress() {
>         return address;
>     }
>     void setAddress(String address) {
>         this.address = address;
>     }
>     String getTelephone() {
>         return telephone;
>     }
>     void setTelephone(String telephone) {
>         this.telephone = telephone;
>     }
> }
> ```
>
> **解题路径**：按面向对象封装思想——3 个私有（此处考纲示例用包级可见）属性 `name/address/telephone`，每个属性配一对 getter/setter；setter 中用 `this` 区分参数与成员变量。考纲此示例使用包级可见成员变量，严格封装应改为 `private`，待人工校对时与教材版本统一。

> 通用方法论：程序设计题遵循「属性→构造方法→读写方法」结构。代码示例取自考纲附录官方示例，**🟢 前须人工逐段编译运行核验并与教材第四版代码风格统一**。

**典型题 1：Java 基本数据类型（第 2 章，识记/简单应用）**

常见考查形式：填空题（「Java 中的基本数据类型包括________」）或简答题。

解题路径：
1. 定位第 2 章「数据类型」——考纲标准分类为四类：布尔型、字符型、整型、浮点型
2. 填空类：需按「四类、按顺序」回答，注意大小写
3. 简答类：每类展开注明具体类型——整型（byte, short, int, long）、浮点型（float, double）、字符型（char）、布尔型（boolean）
4. 常见干扰：将 String 误认为基本数据类型 —— String 是引用类型，不是基本数据类型

**典型题 2：程序改错——数组与循环（第 3/4 章，简单应用）**

常见考查形式：程序改错题（指出带有数组声明 + 循环遍历代码中的错误并改正）。

解题路径：
1. 检查数组声明语法：Java 中 `int a[n] = {...};` 非法，应为 `int[] a = {...};` 或 `int a[] = {...};`
2. 检查循环边界：数组长度 N → 下标 0 至 N-1，循环条件不可用 `<=` 数组长度
3. 检查循环体：`printf`/`println` 的下标引用是否可能越界
4. 改法：去掉声明中的长度数字；修正循环终止条件

**典型题 3：类与对象——属性读写方法（第 5 章，综合应用）**

常见考查形式：程序设计题（按需求定义类，含属性与 getter/setter）。

解题路径：
1. 定义类名——与题目要求一致
2. 声明成员变量——按题目指定类型与个数
3. 编写 getter——`返回类型 getXxx() { return xxx; }`
4. 编写 setter——`void setXxx(类型 参数) { this.xxx = 参数; }`
5. 检查 this 关键字使用——setter 中参数与成员变量同名时用 this 区分

**典型题 4：继承与覆盖（第 5 章，综合应用，本课程重点+难点）**

常见考查形式：程序分析题（给出父子类代码，分析输出结果）或简答题。

解题路径：
1. 识别继承关系——子类 extends 父类
2. 区分隐藏（成员变量）与覆盖（方法），判断多态调用时实际执行哪个版本
3. 查 super 关键字——子类用 `super.xxx` 引用父类成员
4. 检查构造方法链——子类构造方法默认或显式调用 `super()`
5. 常见陷阱：变量隐藏不参与多态；方法覆盖参与多态，运行时根据对象实际类型调度
